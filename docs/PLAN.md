# Plan: Admin Approval for Access Requests

## Уточнение
Пользователь обратил внимание, что в `api.go` **уже существует** готовый эндпоинт `POST /api/request-access`, который делает ровно то, что нужно (отправляет уведомление админу с формой выдачи подписки через `/sub_grant`).

Следовательно, изменения на бэкенде вообще не требуются!

## Итоговый путь реализации (Только Frontend)

### Frontend (`frontend/index.html`)
- Меняем логику в блоке обработки пейволла (когда `!p.has_subscription`).
- Удаляем формирование `adminUrl` и функцию перехода в профиль админа.
- Пишем `onclick` обработчик для кнопки `btn-paywall-contact`, который:
  1. Блокирует кнопку (защита от двойного клика).
  2. Вызывает `await apiFetch('/api/request-access', { method: 'POST' })`.
  3. Меняет текст кнопки на «Запрос отправлен, ожидайте».
  4. (Опционально) Закрывает WebApp через `tg.close()`, чтобы юзер продолжал коммуникацию с ботом.
