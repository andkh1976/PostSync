# PostSync — Telegram & MAX Messenger Bridge

**PostSync** — это высокопроизводительный SaaS-мост для пересылки и синхронизации контента между мессенджерами Telegram и MAX Messenger.

Приложение спроектировано для бесшовного кросспостинга: оно не только перехватывает "свежие" сообщения в режиме реального времени, но и умеет мигрировать исторический контент из Telegram-каналов в MAX, вытягивая даже тяжелые вложения (видео, фото, документы) без перегрузки оперативной памяти.

## Архитектура
Приложение делится на несколько ключевых компонентов:
- **Webhook Gateway** (`api.go`, `telegram.go`, `max.go`): Слушает обновления от Telegram Bot API и MAX Bot API. Маршрутизирует сообщения и управляет командами.
- **SaaS / Multi-tenant Системы**: Управление подписками, изолированные сессии пользователей.
- **MTProto Sync Worker** (`sync_worker.go`): Изолированные клиенты Telegram на базе `gotd/td` для каждого пользователя. Авторизация происходит динамически через Mini App. Используется для парсинга и скачивания исторических сообщений из каналов.
- **Media Streaming Engine** (`upload.go`): Ядро для потоковой (RAM-safe) перекачки файлов напрямую в MAX CDN. Специальный обходчик `customUploadToMax` формирует точный `Content-Length`.
- **Database Repository** (`postgres.go`, `repository.go`): PostgreSQL-хранилище транзакционной привязки пользователей (подписки, сессии), сущностей `TG ID <-> MAX ID` и задач для защиты от дублирования сообщений. SQLite окончательно **упразднён** для совместимости с Multi-Tenant.
- **Frontend Dashboard** (`frontend/`): Встроенный сервер веб-интерфейса (Mini App). Включает в себя авторизацию по номеру, календарь для синхронизации, управление связками.

## Требования
- **Go**: 1.24.0+
- **База данных**: PostgreSQL (Обязательно)
- **Учётные записи**:
  - Токен бота Telegram (`@BotFather`)
  - Токен бота MAX Messenger
  - `API_ID` и `API_HASH` разработчика (для MTProto API работы приложения)
- **Docker**: Опционально, для запуска через `docker-compose`.

## Установка и запуск

### Вариант 1: Запуск в Docker (Рекомендуется)
1. Скопируйте шаблон окружения:
   ```bash
   cp .env.example .env
   ```
2. Заполните `.env` (см. ниже).
3. Запустите оркестратор:
   ```bash
   docker-compose up -d
   ```

### Вариант 2: Локальная сборка
1. `go build -o postsync .`
2. Запустите `./postsync` предварительно настроив `.env` и обеспечив доступ к PostgreSQL.

## Конфигурация (.env)
Обязательно определите следующие переменные в файле `.env` или внутри сервиса деплоя (Coolify / Github Actions):

| Переменная | Описание | Пример |
|------------|----------|--------|
| `TG_TOKEN` | Токен вашего бота в Telegram (BotFather). **(Обязательно)** | `123456:ABC-DEF1...` |
| `MAX_TOKEN` | Токен вашего бота в платформе MAX. **(Обязательно)** | `max-12345-token` |
| `DATABASE_URL` | Строка подключения к PostgreSQL. **(Обязательно)** | `postgres://user:pass@host/db?sslmode=disable` |
| `TELEGRAM_API_ID` | APP ID для подключения к Telegram (MTProto). **(Обязательно для истории)** | `1234567` |
| `TELEGRAM_API_HASH` | Хэш приложения (MTProto). **(Обязательно для истории)** | `abcdef1234567...` |
| `ADMIN_CHAT_ID` | Telegram ID администратора для получения уведомлений и выдачи подписок. | `123456789` |
| `WEBHOOK_URL` | Базовый публичный URL вашего хоста для приёма вебхуков и работы Mini App. | `https://bot.my-domain.com` |
| `WEBHOOK_PORT` | Локальный порт для запуска веб-сервера. | `8443` |
| ~~`ALLOWED_USERS`~~ | **DEPRECATED**. Доступ теперь управляется через БД. | -- |

## Управление доступом (SaaS Биллинг)
Скрытый доступ по белому списку в `.env` отключен. Теперь реализована полноценная база тарифов:
1. Задайте свой Telegram ID в параметр `ADMIN_CHAT_ID` (в `.env`).
2. При регистрации новых пользователей бот в реальном времени пришлет вам уведомление-оповещение.
3. Отправьте боту админскую команду `/sub_grant <UserID> <Дни>` чтобы выдать подписку пользователю. Бот выполнит запрос и откроет статус в БД (без редеплоев и перезапусков!).
4. Вывести список всех пользователей и их статусов можно отправкой команды `/users` напрямую боту.

## Основные функции
1. **Зеркалирование в реальном времени**: Пересылка свежих постов, опросов и текстовых сообщений из Telegram сразу в MAX.
2. **Захват истории (Historical Sync)**: Указав диапазон дат во встроенном Mini App, бот вытащит все посты за этот период из Telegram и перенесет их. Авторизация происходит для каждого клиента индивидуально.
3. **Бесшовная передача медиа**: Фотографии, документы, и самое главное — тяжёлые видео скачиваются через `io.Pipe`, что уберегает бота от вылета из-за нехватки памяти (OOM).
4. **Маппинг форматирования**: Автоматический перевод Telegram-форматирования (MTProto Entities) в Markdown-стандарт MAX.
5. **Предотвращение дубликатов**: При перезапусках синхронизации бот никогда не отправит один и тот же пост дважды.

## Известные ограничения
1. Telegram Bot API имеет лимит в 20 МБ файлов на скачивание. Однако **историческая синхронизация** осуществляется через прямое подключение юзера к MTProto в обход этого лимита — `customUploadToMax` поддерживает загрузку файлов до 2ГБ.
2. Поддержка SQLite полностью прекращена для обеспечения безопасности SaaS-сессий и транзакционности. Используется PostgreSQL.
3. Telegram Premium стикеры или защищенный копирайтом (restricted) контент каналов перенести невозможно.

## Структура проекта
```text
.
├── admin.go            # Валидация прав доступа и ролей чатов (MAX/TG)
├── api.go              # Ядро маршрутизации Mini App API, авторизация сессий
├── bridge.go           # Инфраструктурный слой, безопасность, инициализация
├── format.go           # Конвертеры визуального форматирования текста (HTML/MD)
├── max.go              # Интеграция с SDK и REST MAX
├── telegram.go         # Обработка Telegram, админские команды управления
├── upload.go           # Обходчик для потокового скачивания медиа "на лету"
├── sync_worker.go      # Динамические сессии MTProto для исторического парсинга
├── postgres.go         # SQL-имплементация базы (PG)
├── repository.go       # Универсальный интерфейс БД (CRUD)
├── queue.go            # Буферизация и отложенные задачи пересылки
├── main.go             # Основная точка входа
├── frontend/           # Исходники веб-интерфейса Mini App
└── migrations/         # Файлы `.sql` развертывания таблиц
```
