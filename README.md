# PostSync

**Версия: v1.1.0** · Последнее обновление: 03.04.2026

Сервис односторонней пересылки постов из Telegram-каналов в MAX с поддержкой ретроспективной синхронизации истории.

> **Текущий режим работы:** строго TG → MAX  
> Все операции в обратном направлении (MAX → TG) полностью отключены: новые сообщения не пересылаются, редактирование и удаление постов в MAX не затрагивают Telegram.

## Возможности

- **Кросспостинг TG → MAX** — автоматическая пересылка новых постов из Telegram-канала в MAX-канал
- **Поддержка больших видеофайлов** — пересылка видео до 4 ГБ через локальный Telegram Bot API сервер (требует Telegram Premium у отправителя для файлов > 2 ГБ)
- **Потоковая передача файлов** — видео и документы передаются без загрузки в оперативную память (streaming через io.Pipe)
- **Ретроспективная синхронизация** — скачивание истории TG-канала за произвольный период и публикация в MAX через клиентский протокол MTProto
- **Управление через Mini App** — веб-интерфейс прямо в Telegram и MAX: связки, задачи, прогресс синхронизации, профиль
- **Автозамены** — замена текста, URL и паттернов по регулярным выражениям при пересылке
- **Retry-очередь** — при недоступности API сообщения сохраняются в БД и доставляются после восстановления связи
- **Сохранение форматирования** — жирный, курсив, код, ссылки, зачёркнутый, подчёркнутый
- **Живое прослушивание** — переключение активности потока для каждой связки через Mini App
- **Очистка истории** — сброс внутреннего маппинга сообщений за период для повторной отправки
- SQLite или PostgreSQL для хранения данных

## Установка

### Из бинаря

```bash
chmod +x postsync
./postsync
```

### Docker

```bash
docker run --env-file .env ghcr.io/andkh1976/postsync:latest
```

### Docker Compose (с PostgreSQL и поддержкой больших файлов)

Docker Compose автоматически запускает три сервиса: `bridge`, `postgres` и `telegram-bot-api`.  
Локальный Bot API сервер снимает стандартный лимит 50 МБ и позволяет пересылать видео до 4 ГБ.

```bash
cp .env.example .env
# Обязательно заполните TG_TOKEN, MAX_TOKEN и для большого файла:
# TELEGRAM_API_ID и TELEGRAM_API_HASH — получить на my.telegram.org
docker compose up -d
```

> **Примечание:** `TELEGRAM_API_ID` и `TELEGRAM_API_HASH` нужны только для сервиса `telegram-bot-api` (поддержка файлов > 50 МБ). Если эти переменные не заданы, бот работает в стандартном режиме с ограничением 50 МБ.

### Из исходников

```bash
git clone https://github.com/andkh1976/PostSync.git
cd PostSync
CGO_ENABLED=1 go build -o postsync .
./postsync
```

> **Важно:** `CGO_ENABLED=1` обязателен — проект использует SQLite через CGo. Установите `gcc` и заголовки C: `build-essential` (Debian/Ubuntu) или `base-devel` (Arch).

## Быстрый старт

### 1. Создайте ботов

- **Telegram** — через [@BotFather](https://t.me/BotFather)
- **MAX** — через [dev.max.ru](https://dev.max.ru/docs/chatbots/bots-create)

### 2. Настройте переменные окружения

Скопируйте `.env.example` в `.env` и заполните:

```env
TG_TOKEN=your_telegram_bot_token
MAX_TOKEN=your_max_bot_token

# Для ретроспективной синхронизации (MTProto) — получить на my.telegram.org
TG_APP_ID=12345678
TG_APP_HASH=abcdef1234567890abcdef1234567890
TG_PHONE=+79001234567

# Mini App — URL, по которому доступен веб-интерфейс
MINI_APP_URL=https://your-domain.com:8443/app/

# Хранилище (по умолчанию — SQLite)
# DB_PATH=bridge.db

# Или PostgreSQL
# DATABASE_URL=postgres://user:pass@localhost/postsync
```

### 3. Запустите бота

```bash
./postsync
```

При первом запуске с MTProto бот запросит код подтверждения из Telegram. После успешной авторизации сессия сохраняется в файл `TG_SESSION_FILE` — повторный ввод кода не нужен.

### 4. Свяжите каналы

Откройте Mini App через кнопку в боте и перейдите на вкладку **«Каналы»**.

1. Нажмите **«Добавить связку»**.
2. Выберите Telegram-канал из списка доступных вам каналов.
3. Выберите MAX-чат, с которым хотите синхронизировать публикации.
4. Нажмите **«Сохранить»** — связка создаётся мгновенно через API.

После создания связки посты из Telegram-канала будут автоматически появляться в выбранном MAX-чате. Управлять существующими связками (просмотр, удаление) можно там же в Mini App.

## Mini App

Mini App — основной интерфейс управления PostSync. Открывается прямо внутри Telegram или MAX без перехода в браузер.

### Как открыть

**Через Telegram:**

1. Напишите боту `/start`
2. Нажмите кнопку **⚙️ Открыть панель управления**

Или настройте постоянную кнопку через BotFather:
- `/mybots` → выберите бота → **Bot Settings → Menu Button → Configure menu button**
- URL: `https://your-domain.com:8443/app/`
- Текст: `⚙️ Настройки`

**Через MAX:**

Mini App работает только внутри Telegram (авторизация через `initData`). Из MAX управление через команды бота:
- `/chatid` — узнать ID текущего чата и добавить его в список доступных MAX-каналов
- `/help` — справка по боту

### Разделы

#### 👤 Профиль

- ID пользователя и платформа (Telegram / MAX)
- Имя и username
- Статус подписки

#### 🔗 Активные связки каналов

Список активных связок, созданных через Mini App. Для каждой связки:

- **Направление** — TG → MAX / MAX → TG / оба. Сейчас активен только режим TG → MAX
- **Живое прослушивание** — переключатель активности потока. Выключение останавливает пересылку новых постов без удаления связки

Настройки сохраняются автоматически при изменении.

#### ⏳ Задачи синхронизации

Список запущенных задач ретроспективного скачивания:

- Статус: Ожидание → В процессе → Завершено / Ошибка / Отменена
- Прогресс-бар
- Канал (TG ID → MAX ID) и период
- **⏹ Стоп** — кнопка появляется у активных задач; останавливает синхронизацию между следующими сообщениями

Раздел автообновляется каждые 5 секунд, пока есть активные задачи.

**🗑 Очистить историю** — кнопка в заголовке раздела. Удаляет завершённые, упавшие и отменённые задачи из списка. Активные задачи не затрагивает.

#### 📥 Синхронизация истории

Скачивает историю TG-канала за указанный период и публикует посты в связанный MAX-канал.

> Требует MTProto (переменные `TG_APP_ID` и `TG_APP_HASH`). Без них раздел недоступен.

1. Выберите связку из списка
2. Укажите период в формате `ДД.ММ.ГГГГ` (точки ставятся автоматически при вводе цифр)
3. Нажмите **▶ Синхронизация**

Задача создаётся в фоне — прогресс отслеживается в разделе «Задачи синхронизации».

**🗑 Сбросить историю** — очищает внутренний маппинг ID сообщений за период. Посты в каналах не удаляются. Используйте, если нужно повторно опубликовать уже пересланные посты.

### Авторизация

Mini App использует стандартный механизм Telegram WebApp (`initData`) для проверки подлинности пользователя. Открытие через кнопку в боте происходит автоматически — никаких дополнительных шагов не требуется.

При открытии напрямую через браузер авторизация недоступна — раздел «Профиль» покажет сообщение об ошибке.

## Ретроспективная синхронизация (MTProto)

Для скачивания истории TG-каналов используется Telegram User API (MTProto) — это отдельный клиент, работающий от имени пользователя, а не бота. Стандартный Bot API не предоставляет доступ к истории каналов.

### Настройка

1. Откройте [my.telegram.org](https://my.telegram.org) → **API development tools**
2. Создайте приложение — получите `api_id` и `api_hash`
3. Добавьте в `.env`:

```env
TG_APP_ID=12345678
TG_APP_HASH=abcdef1234567890abcdef1234567890
TG_PHONE=+79001234567
```

4. Перезапустите бота — он запросит код из Telegram для первичной авторизации
5. Сессия сохраняется в `TG_SESSION_FILE` (по умолчанию `tg_session.json`) — повторный вход не нужен

Если `TG_APP_ID` и `TG_APP_HASH` не заданы, sync worker не запускается, а кнопка «Запустить скачивание» в Mini App недоступна.

## Команды бота

Управление PostSync полностью перенесено в Mini App. Текстовые команды сведены к минимуму.

| Команда | Где | Описание |
|---------|-----|----------|
| `/start` | TG или MAX | Запуск бота + кнопка открытия Mini App |
| `/help` | TG или MAX | Инструкция |
| `/chatid` | MAX-канал | Сохранить ID текущего MAX-канала в список для связки |

### Отключённые (legacy) команды

Следующие команды **отключены** (закомментированы в коде с пометкой `// DEPRECATED (Sprint 4 Final)`). Они не обрабатываются ботом и недоступны в меню. Исходный код сохранён для истории.

| Команда | Описание |
|---------|----------|
| `/crosspost` | Список активных связок |
| `/crosspost <TG_ID>` | Начать настройку новой связки через бота |
| `/bridge` | Создать ключ для связки групп |
| `/unbridge` | Удалить связку групп |
| Переслать пост | Показать ID канала или кнопки управления |

Вся настройка — создание связок, управление направлением, ретроспективная синхронизация, автозамены — выполняется исключительно через **Панель управления (Mini App)**.

## Changelog

### v1.1.0 — 03.04.2026

#### Безопасность и корректность направления

- **Отключена синхронизация удаления MAX → TG** — удаление поста в MAX-канале больше не удаляет соответствующее сообщение в Telegram. Система работает строго в одну сторону.
- **Отключена синхронизация редактирования MAX → TG** — изменение постов в MAX не влияет на Telegram. Убран неиспользуемый импорт `tgbotapi` из `max.go`.
- **Подтверждена однонаправленность** — MAX → TG пересылка новых сообщений была отключена ранее; теперь все операции обратного направления (пересылка, редактирование, удаление) полностью деактивированы.

#### Поддержка больших файлов (до 4 ГБ)

- **Добавлен сервис `telegram-bot-api`** в `docker-compose.yml` — официальный локальный Telegram Bot API сервер (образ `aiogram/telegram-bot-api:latest`), снимающий лимит 50 МБ.
- **Добавлен том `tg-api-data`** для сохранения данных локального сервера.
- **Потоковая передача файлов** — `customUploadToMax` переписан с `bytes.Buffer` на `io.Pipe()`: файлы передаются в MAX CDN без загрузки в оперативную память.
- **Прогресс-лог загрузки** — для файлов > 100 МБ в лог выводится прогресс каждые 10%.
- **Таймаут 15 минут** для CDN-загрузки крупных файлов (ранее 5 мин).
- **MAX → TG без буферизации** — при наличии локального Bot API видео/аудио/файлы из MAX отправляются в TG через `FileURL` (сервер сам скачивает файл).
- **Предупреждение для файлов > 2 ГБ** — в лог выводится сообщение о необходимости Telegram Premium у отправителя.
- **Обновлены переменные окружения** — добавлены `TELEGRAM_API_ID` и `TELEGRAM_API_HASH` в `.env.example`.
- **Улучшен лог старта** — при запуске выводится, какой API используется и какой лимит файлов активен.

#### Прочее

- Таймаут `httpClient` увеличен с 5 до 15 минут.
- Улучшены сообщения об ошибках CDN-загрузки: логируются статус-код, тело ответа, имя и размер файла.
