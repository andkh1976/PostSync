# Реализация пейволла в Mini App

## Цель
Реализовать Option A: элегантный экран «Пейволла» (онбординга) внутри Mini App для незарегистрированных пользователей, а также обезопасить все изменяющие состояние API-эндпоинты для предотвращения несанкционированных действий извне.

## Обзор
Этот план направлен на улучшение UX для пользователей без подписки. Вместо пустого дашборда и мертвых кнопок они будут видеть понятный экран с призывом к действию (связаться с менеджером/оплатить подписку). Одновременно с этим на бэкенде закрываются уязвимости, которые технически могли позволить использовать API в обход интерфейса.

## Стек технологий
- Фронтенд: Vanilla JS / HTML / CSS (редактирование `index.html`)
- Бэкенд: Go (редактирование обработчиков в `api.go`)
