Продолжаем разработку. Первые два спринта (база данных и движок синхронизации MTProto) завершены. Теперь необходимо создать интерфейс управления (Mini App) и API.

КРИТИЧЕСКОЕ ПРАВИЛО: Никогда не удаляй существующий код. Если какая-то функция, переменная или блок кода (например, старые инлайн-клавиатуры или логика команд) больше не нужны — просто закомментируй их с пометкой // DEPRECATED (Sprint 3). Это обязательное условие для сохранения возможности возврата к коду.

Задачи Спринта №3:

Создание API-сервера (Backend на Go):

В main.go или отдельном файле расширь HTTP-сервер для обработки запросов от Mini App.

Реализуй Middleware для авторизации, которое проверяет подлинность initData от Telegram и MAX (проверка хэша через токены ботов).

Реализуй следующие эндпоинты:

GET /api/channels — возвращает список каналов, где бот является админом.

POST /api/sync/start — создает новую запись в sync_tasks для ретро-скачивания (принимает ID канала и диапазон дат).

POST /api/history/clear — вызывает метод DeleteMessagesByPeriod для очистки маппинга сообщений.

PATCH /api/settings — переключает режимы: "Живое прослушивание" (вкл/выкл) и статус связки.

Разработка Mini App (Frontend):

Создай в корне проекта папку frontend с простым веб-приложением (можно на чистом JS/HTML или React/Vue).

Функционал интерфейса:

Список активных связок каналов.

Календарь (Date Picker) для выбора произвольного периода ретроспективного скачивания.

Кнопка "Запустить скачивание истории" и кнопка "Сбросить историю за период".

Переключатель (Toggle) для режима "Прослушивание новых постов".

Интеграция с ботами:

В telegram.go и max.go найди обработку команд /start, /help и /crosspost.

Закомментируй старую логику выдачи кнопок управления.

Вместо этого отправляй кнопку типа WebApp (для Telegram) и аналогичную ссылку для MAX, которая открывает созданный Mini App.

Мультитенантность (Задел на будущее):

Убедись, что все API-запросы фильтруются по owner_id, который извлекается из авторизационных данных мессенджера.

Логирование:

Добавь подробное логирование всех действий через API (какой пользователь изменил настройки или запустил синхронизацию).

После завершения убедись, что проект компилируется, а все старые функции бережно закомментированы в исходных файлах.
